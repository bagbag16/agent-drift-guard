# Agent Drift Guard (adg)

**面向多轮 AI 协作的轻量目标漂移护栏。**

和 AI 长对话时,目标会悄悄滑走:问题被换成了更好回答的那个,三轮前的一个"也许"如今
成了既定事实,agent 一路附和着越过了你真正在意的点。adg 是一小组规则,让多轮协作保持
诚实——又不把它做成流程负担。

## 问题

多轮 AI 工作里的漂移很少是吵闹的。它长这样:

- 回答慢慢不再解决最初的问题;
- 三轮前的一个"也许",现在成了承重的事实;
- agent 在迎合选定路径,而不是指出它的毛病;
- 问题定义、方案、执行被搅成一团。

adg 盯的就是这个——稳住问题、保住语义、减少漂移。

## adg 做什么

adg 用五条底线约束一段对话:

1. **三态信息分类。** 一切内容分为 `已确认` / `暂定假设` / `待确认`。高影响内容在确认
   前不得作为既定前提继续扩写——而且分类要能说明**依据**,不只是贴个标签。
2. **高影响改写并排可见。** 改写若会影响目标、结构、命名、规则或约束,就把用户原意与
   建议版本并排展示;确认前,改写版只算 `待确认`。
3. **未定义术语先挂起。** 私有名词、缩写、代称先标 `待确认`,不自行补全语义后再推演。
4. **不以迎合代替判断。** 弱判断不包装成强结论;部分成立要拆成 成立 / 不成立 / 依赖前提;
   证据不足要明指。
5. **一次只处理当前层。** 问题定义、框架、决策、执行、润色分开——不把下一层的决定偷渡
   进这一层。

它还保有一份**轻量主观能动性**:当你选的切口可行、但存在明显更优且更低成本的路径时,
adg 会提醒——说明局限、更优路径、以及为什么更优——但不替你改写目标。

## 何时用、何时别用

**用 adg**:对话跑了好几轮、开始漂移、输入混着目标与修正与新约束,或某个定义不留档就会
失真时。

**别用**:一次性问答、简单编辑、下一步已经清楚且低风险的任务。别在不需要流程的地方搭流程。

## 当任务长过 adg → ACH

adg 刻意做轻。一旦任务需要**持久、可恢复的状态**——熬过交接、在新对话里恢复、追踪任务
如何分叉以及为何分叉——对话里的护栏就不够了。

这正是 adg 的重型演进版要做的:

> **→ [Agent Continuity Harness (ACH)](https://github.com/bagbag16/agent-continuity-harness)**
> —— 正式状态窗口、关键内容外化记录,以及一棵覆盖任务演化的版本树,面向长任务与跨窗口
> 连续性。
